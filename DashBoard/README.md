# Task Management Board

## Tech Stack & Key Choices

-   React (Vite):** Used for fast bundling,and smooth local development.
- **Redux Toolkit:** Used to handle global state management (creating tasks, deleting tasks, and moving tasks between columns).
- **Tailwind CSS:** Used for utility-first styling to achieve a clean and modern layout.
- **React Icons:** Used for lightweight and clean UI icons.

---



## Core Architecture & Approach

1. **Native Drag & Drop (Zero Dependencies):**
   Instead of using heavy external libraries (like `react-beautiful-dnd`), we implemented the **Native HTML5 Drag and Drop API**. This keeps the project bundle size small, highly optimized, and extremely fast.
2. **Smooth Visual Hover States:**
   When a user drags a card over any column, that specific column dynamically changes its look (adds a dashed border and shifts background color). This provides clear visual feedback to the user about where the card will land.
3. **Automated Overdue Alert:**
   The application automatically checks the task due dates. If a task is past its due date and is not completed, a prominent red alert banner appears at the top of the dashboard.

---

## State Management Approach

**Redux Toolkit** was chosen for centralized state management.

Instead of storing separate arrays for each column (To Do, In Progress, Done), **all tasks are stored in a single collection** inside the Redux store. Each task contains a `status` field that determines which column it belongs to.

**Task Entity Structure Example:**
```json
{
"id": "unique-task-id",
"title": "Implement Drag and Drop",
"description": "Create native HTML5 drag and drop handlers",
"status": "todo",
"priority": "High",
"dueDate": "2026-06-20",
"createdAt": "2026-06-17"
}
```

This approach was chosen because a task remains the same entity throughout its lifecycle and only its status changes. Using a single task collection avoids state duplication, simplifies task movement, improves scalability, and aligns closely with real-world database design.

Columns are generated by filtering tasks based on their status rather than maintaining separate state collections for each column.

Benefits:
- Single source of truth
- Easier task movement between columns
- Cleaner and more maintainable state structure
- Better scalability when introducing new statuses
- Simpler drag-and-drop integration
- Database-friendly data modeling



**State Persistence**
Redux state is persisted using Local Storage. This ensures that tasks remain available even after a browser refresh, providing a better user experience and satisfying the assignment requirement for persistence.
